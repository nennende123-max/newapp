| 项目 | 说明 |
|------|------|
| 产品名称 | TruthFi（加密货币交易与理财 Web App） |
| 文档版本 | v1.0 |
| 更新日期 | 2026-05-25 |
| 技术栈 | 前端 Vue 3 + Vite + Vant；主后端 FastAPI（:8000）；辅助 Express+MySQL 矿机服务（:3000） |

---

## 1. 项目概述

### 1.1 产品定位

TruthFi 是一款面向移动端优先的加密货币综合平台，提供行情浏览、现货/合约交易、理财申购、云挖矿、Launchpad（IDO）、预测市场、钱包资产管理及邀请返佣等能力。界面风格参考主流交易所（黑金主题），支持中英文切换。

### 1.2 目标用户

- 持有 Web3 钱包（以太坊地址）的加密货币用户
- 需要一站式查看行情、交易、理财与参与活动的用户

### 1.3 核心价值

- **一站式**：首页聚合行情、快捷入口与平台透明度数据
- **实时行情**：对接 Binance WebSocket / REST，K 线与盘口实时更新
- **模拟交易闭环**：钱包签名登录 → 充值/交易 → 资产与订单可查（当前多为 Mock/本地持久化）

## 2. 系统架构

### 2.1 部署与访问

| 组件 | 端口 | 说明 |
|------|------|------|
| 前端 Vite Dev | 5173 | `npm run dev`，`/api` 代理到 8000 |
| FastAPI 主后端 | 8000 | `uvicorn main:app --reload` |
| Express 矿机服务 | 3000 | `server/server.js`，MySQL `crypto_exchange` |
| Binance 行情 | 外部 | 直连或经 Vite `/ws` 代理 |

### 2.2 前端结构

- **路由**：`src/router/index.js`（约 20+ 页面）
- **状态**：Pinia（`assets`、`market`、`trade` 等）
- **API 封装**：`src/api/*` + `src/utils/request.js`（JWT 自动携带）
- **国际化**：`vue-i18n`（中/英，`src/main.js` 内文案）

### 2.3 后端结构

- **认证**：`POST /api/v1/auth/connect`（钱包签名 + JWT）
- **资产**：`app/api/endpoints/assets.py`
- **现货**：`app/api/endpoints/trade.py`
- **合约**：`app/api/endpoints/futures.py`
- **行情**：`app/api/endpoints/market.py` + `market_service`（Binance 流）
- **数据层**：`app/db/mock.py`（JSON/内存 Mock）、SQLite（K 线、认购表结构已初始化）

## 6. 数据与业务规则

### 6.1 账户权益

```
账户权益 = 可用余额 + 冻结保证金 + 合约未实现盈亏
```

### 6.2 手续费

| 类型 | 费率 |
|------|------|
| 现货 | 0.1%（`BASE_FEE_RATE`） |
| 合约 | 0.04%（`FUTURES_FEE_RATE`） |

### 6.3 支持币种（资产侧）

USDT、BTC、ETH、BNB、SOL、DOGE、TRX、BEAT、AIC 等；价格来自 Mock 或 Binance。

### 6.4 订单状态

`NEW` → `PARTIALLY_FILLED` / `FILLED` / `CANCELLED`（现货）；合约含挂单冻结保证金逻辑。

### 6.5 持久化

| 数据 | 存储 |
|------|------|
| JWT、钱包连接 | localStorage |
| Mock 用户资产/订单/持仓 | JSON 文件 + 内存（`app/db/mock.py`） |
| K 线历史 | SQLite `klines` |
| 认购项目/订单 | SQLite 表已初始化，业务 API 未完全暴露 |
| 预测市场订单 | localStorage |
| 谷歌验证/资金密码 | localStorage |

## 7. 非功能需求

### 7.1 性能

- 首屏行情应在 WS 连接后 3s 内有价格更新
- 下单接口超时 10s（axios `timeout`）
- K 线图表支持多周期：1m / 5m / 15m / 1h / 4h / 1D

### 7.2 安全

- 生产环境必须配置 `JWT_SECRET_KEY` 环境变量
- CSP 限制脚本与连接源（`index.html` + Vite headers）
- 敏感操作（提现、交易）需 JWT；资金密码/2FA 为前端本地校验（待接后端）

### 7.3 兼容性

- 移动端优先，安全区域 `safe-area-inset`
- 中英文 i18n 覆盖主要页面

### 7.4 可用性

- 未连接钱包时，交易/认购等操作引导连接
- 加载态、空状态、错误 Toast 提示
